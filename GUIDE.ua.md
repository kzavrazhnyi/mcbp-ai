## Головний принцип

Модель сесії впливає лише на **оркестратора** — тріаж, диспатч і синтез звітів. Кожен агент в
`.claude/agents/` має власний `model:` у frontmatter і використовує саме його незалежно від
моделі сесії. Тому дешева модель сесії не послаблює агентів пайплайна, а дорога — не «прокачує» їх.

Режими дозволів (Shift+Tab перемикає):

- **plan** — спочатку план, правки після підтвердження. Для багатофайлових фіч і refactor-чартерів.
- **default** — питає дозвіл на кожну правку. Для обережних разових змін.
- **acceptEdits / auto** — правки приймаються автоматично. Для рутинної роботи всередині репо.

## Дефолтна модель сесії

Цей конфіг шипить `settings.json` з `model: sonnet` — правильний дефолт для тарифу
`pro-20`. Змінюй на сесію через `/model`, коли ситуація нижче цього вимагає; моделі самих
агентів лишаються як у мапі в `LIMITS.md`.

## Ситуації та економія лімітів

- **Лишайся на `sonnet` як моделі сесії.** На цьому тарифі немає `opusplan` — plan mode мовчки
  палив би Opus-ліміт Pro. Plan mode усе ще дає рев'ю плану — використовуй лише для фіч
  на >5 файлів і refactor-чартерів.
- **Quality-гейти йдуть послідовно** (reviewer → tester → security-scanner), один пайплайн за раз.
- **Usage вище ~70%?** Відклади L-пайплайни (повна фіча + гейт); ганяй S/M задачі або дочекайся скидання.
- **`/clear` між незв'язаними задачами, `/compact` між фічами** — довга сесія перечитує всю
  історію на кожному кроці.
- **Не передиспатчуй те, що агент уже відповів** — використовуй його звіт; повторні прогони —
  тихий витік бюджету.

| Ситуація | Що робити |
|---|---|
| Велика фіча (>5 файлів) | Plan mode, далі auto на виконання |
| Багфікс / дрібна зміна | Auto, без plan mode |
| Багато дрібних питань підряд | Лишайся на sonnet, `/clear` між темами |
| Usage > 70% перед великою задачею | Відклади або розбий на S/M шматки |
| Довга сесія, контекст росте | `/compact` на межі задач — ніколи посеред задачі |

## Що куди копіюється

Ці файли скопійовані з виводу генератора `projects/mcbp-ai/`. `README.md` конфігу
документує сам конфіг і лишається в архіві генератора — сюди він не копіюється. Усе, що БУЛО
скопійовано (`.claude/`, `CLAUDE.md`, `.mcp.json`, `LIMITS.md`, цей гайд, опційні
`ROADMAP.md`/`STATE.md`/`lessons.md`), ігнорується блоком "Claude Code configuration" у
`.gitignore` — тому два конфіги ніколи не конфліктують в одному репо. Якщо команда вирішить
трекати щось із цього в git — видали відповідний рядок у `.gitignore`.
